## [[Blog Post -- Section 13 - Gas Town Plugins]]


How are [[Term -- Gas Town Plugin|plugins]] triggered within a workflow?
?
Any workflow can contain "run plugins" steps — they're just steps in a [[Term -- Molecule|molecule]].
<!--GUID:27f13f7bf31fd-->

<!--MEADOW_SR_GUID:630f95b1ddfc6-->


How are [[Work Level -- Town|Town-level]] [[Term -- Gas Town Plugin|plugins]] run?
?
[[Term -- The Deacon|The Deacon]] runs them as part of its [[Term -- Patrol|patrol]], but hands the actual work off to [[Term -- Dogs|Dogs]] so they can run for essentially unlimited time.
<!--GUID:5d0aaa22d01b4-->

<!--MEADOW_SR_GUID:259188da20f27-->


Why are [[Term -- Dogs|Dogs]] used to run [[Work Level -- Town|Town-level]] [[Term -- Gas Town Plugin|plugins]] instead of [[Term -- The Deacon|The Deacon]] running them directly?
?
So plugins can run for essentially unlimited time without blocking [[Term -- The Deacon|The Deacon]]'s [[Term -- Patrol|patrol]] loop and also messing with the [[Term -- Town Mail|Town Mail]]'s latency
<!--GUID:322362019876b-->
